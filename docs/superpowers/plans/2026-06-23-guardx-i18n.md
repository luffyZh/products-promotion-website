# GuardX 多语言（方案A）Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 将 `GuardX` 页面按“章节块”方式接入 i18n（zh-CN/en/ar），并补齐运行时文案（title/下载名/弹窗/aria-label），确保 `ar` 下 RTL 展示正常。

**Architecture:** 使用 `react-i18next` 的 `t()` + `Trans` 完成块级翻译与少量富文本插槽；将 GuardX 的静态结构（feature/scenario/painpoint）抽象为 key 列表驱动渲染；语言不进 URL，保存在 localStorage。

**Tech Stack:** React + TypeScript + react-i18next + i18next + TailwindCSS

- [ ] **Step 2: 运行时文案改造**
  - `document.title` 使用 `t('guardx.meta.title')`
